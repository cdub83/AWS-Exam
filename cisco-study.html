IPv4 Subnetting Quick Reference

Subnet Math

  • Hosts per subnet = 2^h − 2 (h = host bits)
  • Subnets = 2^s (s = borrowed subnet bits)
  • /24 → 254 hosts; /25 → 126; /26 → 62; /27 → 30; /28 → 14; /29 → 6; /30 → 2
  • Broadcast = last address of subnet block
  • Network = first address; router usually .1

Private Ranges (RFC 1918)

  • 10.0.0.0/8 — Class A private
  • 172.16.0.0/12 — Class B private (172.16–172.31)
  • 192.168.0.0/16 — Class C private
  • 127.0.0.0/8 — loopback
  • 169.254.0.0/16 — APIPA (link-local)

IPv6 Basics

  • 128-bit address, colon-hex notation
  • ::1 — loopback
  • FE80::/10 — link-local (mandatory on every interface)
  • FF00::/8 — multicast
  • 2000::/3 — global unicast (public)
  • EUI-64: uses MAC address to auto-generate interface ID
⚠ Exam Trap
TCP guarantees delivery with ACKs, sequencing, and retransmission. UDP does not — it is connectionless and used when speed matters more than reliability (DNS, VoIP, streaming). Confusing these on scenario questions costs marks.
Ethernet Fundamentals

MAC Addresses

  • 48-bit (6-byte) burned-in address (BIA)
  • First 3 bytes = OUI (vendor); last 3 = device identifier
  • FF:FF:FF:FF:FF:FF = broadcast
  • Switches learn MACs from source frames and populate CAM table
  • Unknown unicast = flooded out all ports except source

Half vs Full Duplex

  • Half duplex → CSMA/CD collision detection (hubs)
  • Full duplex → no collisions; both directions simultaneously
  • Modern switches always operate in full duplex
  • Auto-negotiation mismatch (one side full, other half) causes errors

VLANs & Trunking

VLAN Basics

  • VLANs segment broadcast domains at Layer 2
  • Default VLAN = VLAN 1 (avoid using it — security risk)
  • Native VLAN: untagged frames on a trunk (default VLAN 1; must match both ends)
  • Access port: carries one VLAN, no tags
  • Trunk port: carries multiple VLANs using 802.1Q tagging

Inter-VLAN Routing

  • Router-on-a-Stick: single physical link, subinterfaces per VLAN
  • Layer 3 Switch: SVIs (switched virtual interfaces) per VLAN
  • SVI: virtual interface on L3 switch — acts as default gateway for VLAN
  • L3 switch requires ip routing command

VTP (VLAN Trunking Protocol)

  • Server: creates/modifies/deletes VLANs, sends updates
  • Client: receives and applies updates, cannot modify
  • Transparent: local VLANs only, forwards updates
  • VTP revision number: higher wins — joining a rogue switch can wipe VLANs
  • Best practice: use VTP transparent or disable VTP
⚠ Exam Trap
Native VLAN must match on both ends of a trunk. A mismatch causes a CDP warning and traffic from the mismatched VLAN crosses silently into the wrong VLAN — a VLAN hopping vector.

STP Port Roles

  • Root Port (RP): best path toward root bridge (non-root switches)
  • Designated Port (DP): best port on each segment toward root
  • Blocked / Alternate: prevents loops
  • Root bridge: lowest Bridge ID (priority + MAC). Default priority = 32768
  • Lowest Bridge ID wins root election

STP Security

  • PortFast: skip listening/learning on access ports (immediate forwarding)
  • BPDU Guard: shut port if BPDU received (protects access ports)
  • Root Guard: prevents inferior switch from becoming root on a port
  • Loop Guard: prevents unidirectional link failures from creating loops
EtherChannel

EtherChannel Protocols

  • LACP (802.3ad): IEEE standard; Active/Passive modes
  • PAgP: Cisco proprietary; Desirable/Auto modes
  • Static: "on" mode — no negotiation (both ends must be "on")
  • Up to 8 active links; load-balance by src/dst MAC, IP, or port
  • All ports must match: speed, duplex, VLAN config, trunk settings
Wireless LAN (802.11)

Standards

  • 802.11a: 5 GHz, up to 54 Mbps
  • 802.11b: 2.4 GHz, up to 11 Mbps
  • 802.11g: 2.4 GHz, up to 54 Mbps
  • 802.11n (Wi-Fi 4): 2.4/5 GHz, up to 600 Mbps (MIMO)
  • 802.11ac (Wi-Fi 5): 5 GHz, up to 3.5 Gbps (MU-MIMO)
  • 802.11ax (Wi-Fi 6): 2.4/5/6 GHz, OFDMA, up to 9.6 Gbps

Wireless Architecture

  • Autonomous AP: standalone, self-configured
  • Lightweight AP (LWAP): managed by WLC via CAPWAP
  • CAPWAP: UDP 5246 (control), 5247 (data)
  • WLC centrally configures SSIDs, channels, power
  • BSS: single AP cell; ESS: multiple APs, same SSID

Routing Fundamentals

Administrative Distance

  • Connected: 0
  • Static: 1 (floating static: higher AD for backup)
  • EIGRP Summary: 5
  • eBGP: 20
  • EIGRP internal: 90
  • OSPF: 110
  • RIP: 120
  • EIGRP external: 170
  • iBGP: 200

Static Routes

  • ip route network mask {next-hop | exit-if}
  • Default route: 0.0.0.0 0.0.0.0
  • Floating static: set AD higher than dynamic protocol
  • Recursive lookup: router must resolve next-hop to exit interface
  • Directly connected static (exit-if): faster but may have proxy ARP issues

OSPF Basics (CCNA)

  • Link-state, Dijkstra SPF algorithm
  • Uses areas — Area 0 = backbone (mandatory)
  • Metric: cost = 10^8 / bandwidth
  • DR/BDR election on multi-access networks (highest priority wins, then highest Router ID)
  • Hello: 10 s (broadcast); Dead: 40 s. Must match on segment.

NAT / PAT

NAT Types

  • Static NAT: one-to-one permanent mapping
  • Dynamic NAT: pool of public IPs, first-come first-served
  • PAT (Overload): many-to-one using port numbers (most common)
  • Inside local: private IP of internal host
  • Inside global: public IP seen by internet (after NAT)
  • Outside global: public IP of destination server

DHCP

  • DORA: Discover → Offer → Request → Acknowledge
  • DHCP server can be on separate subnet with ip helper-address
  • ip helper-address configured on router interface toward clients
  • DHCP snooping: validates DHCP messages; trusted vs untrusted ports

NTP

  • Stratum 0: reference clock (GPS, atomic); Stratum 1: server synced to stratum 0
  • Cisco default: ntp server <ip>
  • Critical for: syslog timestamps, certificate validity, Kerberos auth
Access Control Lists (ACLs)

ACL Types

  • Standard (1–99, 1300–1999): source IP only; place CLOSE to destination
  • Extended (100–199, 2000–2699): src/dst IP, protocol, port; place CLOSE to source
  • Named ACLs: can insert/delete individual entries with sequence numbers
  • Implicit deny all at end of every ACL
  • Process top-to-bottom; first match wins

Wildcard Masks

  • Inverse of subnet mask: 0 = must match, 1 = don't care
  • /24 → wildcard 0.0.0.255
  • /16 → wildcard 0.0.255.255
  • Any host: host 10.1.1.1 or 10.1.1.1 0.0.0.0
  • Any: any or 0.0.0.0 255.255.255.255
⚠ Exam Trap
Standard ACLs should be placed close to the destination (they can only match source IP and would block too much if placed at the source). Extended ACLs should be placed close to the source to stop unwanted traffic early.
Layer 2 Security

Port Security

  • Limits MAC addresses per port
  • Restrict: drops, logs, port stays up
  • Protect: drops silently, no log
  • Shutdown: err-disables port (default)
  • Sticky: dynamically learned MACs saved to running config

DHCP Snooping / DAI

  • DHCP snooping: blocks rogue DHCP servers on untrusted ports
  • Builds binding table: MAC, IP, port, VLAN
  • Dynamic ARP Inspection (DAI): validates ARP against DHCP snooping table
  • DAI prevents ARP poisoning / MITM attacks
  • Uplink to DHCP server = trusted port

EIGRP (Enhanced IGRP)

EIGRP Metrics & Feasibility

  • Composite metric: K1×BW + K3×Delay (default; K1=K3=1, others=0)
  • Successor: best path (lowest FD — Feasible Distance)
  • Feasible Successor: backup path; RD < FD of current successor (Feasibility Condition)
  • If FC met → instant failover to FS (no query sent)
  • If FC not met → DUAL query process (slower convergence)

EIGRP Named Mode

  • Single config block: router eigrp NAME
  • Supports IPv4 and IPv6 in same process
  • Address families: address-family ipv4 unicast autonomous-system X
  • Per-interface settings under af-interface
  • Recommended over classic (numbered) mode for new deployments

EIGRP Stub & Variance

  • Stub routing: spoke advertises only connected/summary; hub won't query stub
  • Stub types: connected, summary, static, redistributed, receive-only
  • Variance: load-balance over unequal-cost paths (variance multiplier × successor metric)
  • EIGRP summarization: creates null0 summary route to prevent routing loops
OSPF Multi-Area

LSA Types

  • Type 1: Router LSA — intra-area links (every router)
  • Type 2: Network LSA — DR-generated for multi-access segment
  • Type 3: Summary LSA — ABR summarizes inter-area routes
  • Type 4: ASBR Summary — ABR advertises ASBR reachability
  • Type 5: External LSA — ASBR redistributes external routes (E1/E2)
  • Type 7: NSSA External — for NSSA areas (converted to Type 5 at ABR)

OSPF Special Areas

  • Stub: no Type 5; default route injected by ABR
  • Totally Stubby: no Type 3, 4, or 5; only default route (Cisco proprietary)
  • NSSA: allows redistribution of external routes as Type 7; blocks Type 5
  • Totally NSSA: no Type 3 either; external routes as Type 7 (Cisco)
  • Backbone area (Area 0): all areas must connect to it directly or via virtual link

OSPF Metric Tuning

  • Cost = reference BW / interface BW
  • Default reference BW: 100 Mbps — change with auto-cost reference-bandwidth
  • On Gigabit links, all have cost 1 unless reference BW is increased (e.g., 1000 or 10000)
  • Summarization at ABR: area X range; at ASBR: summary-address
  • Virtual link: connects discontiguous Area 0 through transit area
⚠ Exam Trap
OSPF E2 (default for external routes) uses the external metric only — the cost to reach the ASBR is ignored. E1 adds the internal cost to reach the ASBR. When two ASBR paths exist with different internal costs, E1 will prefer the closer ASBR; E2 will not.

BGP Essentials

  • iBGP: same AS; TCP 179; requires full mesh (or RR/confederation)
  • eBGP: different AS; next-hop changes on eBGP peering by default
  • next-hop-self: iBGP peers set themselves as next-hop for eBGP routes
  • Route Reflector (RR): solves iBGP full-mesh; RR reflects routes between clients
  • Synchronization: off by default in modern IOS; required trust between IGP and BGP

BGP Route Control

  • Prefix lists: match specific prefixes with exact or range matching
  • AS_PATH access lists: regex matching on AS_PATH string
  • Route maps: match + set; can modify attributes in/out
  • BGP communities: tag routes for policy; 32-bit value (AA:NN format)
  • Well-known communities: no-export, no-advertise, local-AS
Route Redistribution & Path Control

Redistribution

  • Redistribute between protocols using redistribute command under target protocol
  • Must set a seed metric when redistributing into EIGRP (default metric or per-command)
  • Into OSPF: default metric 20 (E2); specify with metric or default-metric
  • Route tags: mark redistributed routes to prevent re-redistribution loops
  • Mutual redistribution (two-way) requires careful loop prevention

Policy-Based Routing (PBR)

  • Override routing table decisions based on match criteria (ACL, length)
  • Applied inbound on interface with ip policy route-map
  • set ip next-hop: forward to specific next-hop
  • set interface: forward out specific interface
  • Local PBR: ip local policy route-map (for router-generated traffic)

IP SLA & Tracking

  • IP SLA: measures network performance (ICMP echo, UDP jitter, TCP connect)
  • Track object: monitors IP SLA state (up/down)
  • Floating static route: track object + higher AD → automatically fails over
  • PBR + track: change policy when primary path fails
ENARSI Domain 2 — VPN Technologies
20% of exam
▼

DMVPN Phases

  • Phase 1: hub-and-spoke only; spokes never communicate directly
  • Phase 2: spoke-to-spoke via NHRP resolution; spoke routes must not be summarized at hub
  • Phase 3: spoke-to-spoke with NHRP redirect; hub summarizes (scalable); no split-horizon issue
  • mGRE + NHRP + IPsec = DMVPN components
  • NHRP: Next Hop Resolution Protocol — resolves tunnel IP to physical IP

MPLS L3VPN Concepts

  • CE-PE-P-PE-CE architecture; P routers only see MPLS labels
  • VRF: separate routing table per customer on PE router
  • RD (Route Distinguisher): makes overlapping customer prefixes unique in BGP
  • RT (Route Target): import/export policies control VRF route leaking
  • MP-BGP carries VPNv4 routes between PE routers

IPsec Fundamentals

  • IKE Phase 1: establishes secure channel (ISAKMP SA); Main or Aggressive mode
  • IKE Phase 2: negotiates IPsec SA (data protection parameters)
  • ESP: encrypts payload + authenticates; IP protocol 50
  • AH: authenticates only (no encryption); IP protocol 51
  • Transport mode: encrypts payload only; Tunnel mode: encrypts entire IP packet
ENARSI Domains 3 & 4 — Infrastructure Security & Services
40% of exam
▼

uRPF (Unicast RPF)

  • Drops packets where source IP has no valid return path (anti-spoofing)
  • Strict mode: packet must arrive on best-path interface back to source
  • Loose mode: source IP just needs to exist in routing table (any interface)
  • Loose mode safer for asymmetric routing environments

CoPP (Control Plane Policing)

  • Protects router CPU from traffic directed to control plane
  • Uses MQC (Modular QoS CLI): class-maps + policy-maps
  • Applied to control plane with service-policy input under control-plane
  • Prioritize: routing protocols, SNMP, management SSH over bulk traffic

Device Management & QoS

  • SNMPv3 provides authentication (SHA) and encryption (AES/DES) — use over v1/v2c
  • DSCP: 6-bit field in IP header for QoS marking (EF=46 for voice; AF classes for data)
  • QoS models: IntServ (RSVP per-flow), DiffServ (class-based, scalable)
  • Queuing: CBWFQ/LLQ; LLQ = strict priority queue for voice traffic
IPv6 Routing

OSPFv3

  • Runs over IPv6; uses IPv6 link-local addresses for neighbor relationships
  • Supports both IPv4 and IPv6 with address families (IOS XE)
  • Router IDs still 32-bit (IPv4 format)
  • LSA types 8 (Link LSA) and 9 (Intra-Area Prefix LSA) added vs OSPFv2

EIGRPv6

  • Same DUAL algorithm as IPv4 EIGRP
  • Uses IPv6 link-local addresses for neighbor relationships
  • Must enable with ipv6 router eigrp X and no shutdown
  • Per-interface activation: ipv6 eigrp X under interface

SCOR Domain 1 — Security Concepts
25% of exam
▼

CIA Triad

  • Confidentiality: prevent unauthorized disclosure (encryption, ACLs)
  • Integrity: prevent unauthorized modification (hashing, digital signatures)
  • Availability: ensure authorized access (redundancy, DDoS protection)

Common Attack Types

  • Phishing: email-based social engineering to steal credentials
  • Spear Phishing: targeted phishing against specific individuals
  • MITM: intercepts communication between two parties
  • SQL Injection: malicious SQL in web input fields
  • XSS: inject scripts into web pages viewed by other users
  • DDoS: overwhelm resources; volumetric / protocol / application layer
  • Buffer overflow: overwrite memory to execute arbitrary code

Cryptography Fundamentals

  • Symmetric: same key for encrypt/decrypt (AES, 3DES); fast, scalable key distribution problem
  • Asymmetric: key pair — public encrypts, private decrypts (RSA, ECDSA); slower, solves key exchange
  • Hash: one-way (SHA-256, SHA-384, MD5); integrity verification
  • HMAC: hash + shared secret → message authentication
  • PKI: Certificate Authorities issue X.509 certs; chain of trust
Security Models & Frameworks

Defense in Depth

  • Multiple overlapping security layers — no single point of failure
  • Layers: perimeter firewall → IPS → network segmentation → host AV → data encryption
  • Zero Trust: "never trust, always verify"; no implicit trust based on network location
  • Principle of Least Privilege: grant minimum access required

CVSS & Vulnerability Scoring

  • CVSS (Common Vulnerability Scoring System): 0–10 scale
  • Critical: 9.0–10.0; High: 7.0–8.9; Medium: 4.0–6.9; Low: 0.1–3.9
  • CVE: Common Vulnerabilities and Exposures — unique ID per vulnerability
  • NVD (National Vulnerability Database): public CVE repository

Firewall Zones & Policies

ASA Security Levels

  • 0–100 scale; higher = more trusted
  • Outside: 0; Inside: 100; DMZ: typically 50
  • Traffic flows freely from higher to lower security level by default
  • Traffic from lower to higher requires explicit ACL permit
  • same-security-traffic permit: allows inter-interface traffic at same level

IPS vs IDS

  • IDS (Intrusion Detection): passive; monitors copy of traffic; alerts only
  • IPS (Intrusion Prevention): inline; can drop malicious traffic in real time
  • Signature-based: matches known attack patterns
  • Anomaly-based: detects deviations from baseline (behavioral)
  • False positive: legitimate traffic blocked; False negative: attack not detected

Network VPN Technologies

  • AnyConnect SSL VPN: remote access; client-based; TLS/DTLS
  • Clientless SSL VPN: browser-based portal; no client required
  • IPsec IKEv2: site-to-site or remote access; faster IKEv2 negotiation
  • FlexVPN: IKEv2-based framework; hub-and-spoke or point-to-point
  • SSL VPN uses TCP (reliable but higher overhead); DTLS uses UDP (better for real-time)
⚠ Exam Trap
On Cisco ASA, the security level determines traffic flow by default, but ACLs always override security levels. An ACL on the outside interface denying traffic takes precedence over the security level difference. Traffic inspection state is maintained per connection.

Cloud Shared Responsibility

  • Provider responsible: physical hardware, hypervisor, facilities
  • Customer responsible: data, identities, applications, OS (IaaS), network config
  • SaaS: provider manages nearly everything; customer manages data and access
  • PaaS: customer manages apps and data; provider manages runtime/OS
  • IaaS: customer manages OS, middleware, apps, data; provider manages infra

Cisco Umbrella

  • Cloud-based DNS security and Secure Web Gateway
  • Blocks malicious domains at DNS resolution layer (before TCP connection)
  • Uses Anycast DNS (208.67.222.222 / 208.67.220.220)
  • Roaming client for protection off-network
  • Investigate: threat intelligence platform; shows domain relationships

Cisco Email Security (ESA)

  • Protects against spam, phishing, malware in email
  • Inbound: anti-spam, anti-virus, Advanced Malware Protection (AMP), DLP
  • Outbound: DLP (Data Loss Prevention), email encryption
  • SPF, DKIM, DMARC: email authentication to prevent spoofing
  • SBRS (SenderBase Reputation Score): sender reputation filtering

Cisco Web Security (WSA / SWG)

  • Proxy-based; controls web traffic by category, reputation, and content
  • Explicit proxy: client configured to use WSA
  • Transparent proxy: traffic redirected via WCCP or policy routing
  • SSL/TLS decryption: man-in-the-middle to inspect HTTPS
  • AMP for Endpoints integration: file reputation and sandboxing
SCOR Domains 5 & 6 — Endpoint Protection & Secure Network Access
25% of exam
▼

Cisco AMP for Endpoints (Secure Endpoint)

  • Advanced Malware Protection: continuous file monitoring beyond initial verdict
  • File reputation: cloud lookup against known good/bad hash database
  • Behavioral analysis: detects malware by behavior, not just signature
  • Device trajectory: timeline of file activity on endpoint for forensics
  • Retrospective detection: alerts if a previously "good" file later turns malicious

Cisco ISE (Identity Services Engine)

  • AAA server: Authentication, Authorization, Accounting
  • RADIUS protocol for 802.1X wired/wireless authentication
  • Posture assessment: checks endpoint compliance before granting access
  • Profiling: identifies device type based on attributes (DHCP, HTTP, etc.)
  • Guest access, BYOD onboarding, device registration

802.1X Authentication Flow

  • Supplicant (endpoint) ↔ Authenticator (switch/AP) ↔ Authentication Server (ISE/RADIUS)
  • EAP methods: EAP-TLS (cert-based, most secure), PEAP (password + server cert), EAP-FAST
  • MAB (MAC Authentication Bypass): for devices without 802.1X supplicant
  • dACL (downloadable ACL): ISE sends ACL to switch post-authentication
  • VLAN assignment: ISE assigns VLAN based on auth result

TrustSec & MACsec

  • TrustSec: SGT (Security Group Tag) assigned at ingress; policy enforced at egress
  • SGTs replace IP-based ACLs with identity-based segmentation
  • MACsec (802.1AE): Layer 2 hop-by-hop encryption
  • MACsec encrypts all frames between directly connected devices
  • Key exchange: MKA (MACsec Key Agreement) protocol
✓ Key Concept
Cisco ISE, TrustSec, and Cisco DNA Center work together: ISE authenticates and assigns SGTs, TrustSec enforces SGT-based policy across the network, and DNA Center provides centralized visibility and automation. This is the core of Cisco Software-Defined Access (SD-Access).
